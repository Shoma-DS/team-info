## 概要

### このスキルでできること

✅ **アカウント連動**: アカウントごとのブランドトーンを自動適用
✅ **短尺/長尺選択**: 5-15分の短尺、40-120分の長尺を選択可能
✅ **3つのファイル読み込み**: 構成の見本、アカウント情報、ネタの台本を読み込み
✅ **台本の要約**: 参考台本を自動で要約し、構成・文体・内容を学習
✅ **段階的執筆**: 一章ずつ執筆し、章ごとに通知しながら自動進行
✅ **セッション管理**: 途中で中断しても、後から続きを再開可能
✅ **一貫性の維持**: 毎回、2つの要約と章構成を読み込んで執筆
✅ **音声化向け分割**: 長文を句読点単位で分割し、TTSエラーを回避

### 新バージョン（v2.0）の主な改善点

🆕 **短尺/長尺の選択機能**
🆕 **見本台本の自動要約作成**
🆕 **一章ごとの自動進行通知**
🆕 **セッション管理（中断・再開可能）**
🆕 **ネタの種を毎回読み込み（一貫性向上）**

## クイックスタート

### 最速で台本を作成する

```
台本を作成してください。
アカウント: sleep_travel
テーマ: エジプト神話
```

↓

スキルが以下の質問をします：

1. **「短尺（5-15分）と長尺（40-120分）、どちらにしますか？」**
   → 例: 「長尺」と回答

2. **「台本構成の見本を選択してください」**（リスト表示）
   → 例: 「sleep_travel_greek_mythology」と回答

3. **「台本のネタとする台本を選択してください」**（リスト表示）
   → 例: 「sleep_travel_universe_01」と回答
   → ※構成の見本と同じでもOK

4. **「3つのファイルを読み込み、要約を作成しました」**（自動処理）

5. **「章構成でよろしいですか？」**（構成案を提示）
   → 例: 「はい」と回答

6. **「第1章が完成しました。次に進みます。」**
   → 章ごとの確認待ちはせず、自動で次章へ進行

7. **「全ての章が完成しました！」**
   → 最終台本のパスが表示されます

## 詳細な使い方

### ステップ1: アカウント選択

利用可能なアカウント一覧が表示されます。

**例:**
```
以下のアカウントが利用可能です:
1. sleep_travel（寝ながらトラベル - 睡眠導入型）
2. example_account（サンプルアカウント）

どのアカウントを使用しますか？
```

→ アカウント名を答えてください（例: `sleep_travel`）

### ステップ2: 短尺/長尺の選択

**短尺（5-15分）**: SNS向け、教育コンテンツ、テンポが速い
**長尺（40-120分）**: 睡眠導入、深掘りコンテンツ、ゆったり

**例:**
```
短尺（5-15分）と長尺（40-120分）、どちらにしますか？
- 短尺: SNS、YouTube短編、教育コンテンツ
- 長尺: 睡眠導入、長編朗読、深い教育コンテンツ
```

→ 「短尺」または「長尺」と答えてください

**迷ったら**: [短尺と長尺の選び方](#短尺と長尺の選び方) を参照

### ステップ3: 構成の見本となる台本の選択

構成パターン、文体、リズムを参考にする台本を選びます。

**例:**
```
台本構成の見本を選択してください:
1. sleep_travel_greek_mythology（ギリシャ神話編）
2. example_sample_script（サンプル台本）

どの台本を構成の見本にしますか？
```

→ ファイル名を答えてください（例: `sleep_travel_greek_mythology`）

### ステップ4: 内容のネタとなる台本の選択

内容、情報の提示方法を参考にする台本を選びます。

**例:**
```
台本のネタとする台本を選択してください:
1. sleep_travel_greek_mythology（ギリシャ神話編）
2. sleep_travel_universe_01（宇宙の旅編）

どの台本をネタの参考にしますか？
```

→ ファイル名を答えてください（例: `sleep_travel_universe_01`）

**ポイント**: 構成の見本とネタの台本は同じでも異なっても構いません。
- 同じ: 構成も内容も同じジャンルで統一したい場合
- 異なる: 例えば「ギリシャ神話の構成」で「宇宙の内容」を書く場合

### ステップ5: 台本の要約作成（自動）

スキルが自動的に2つの台本を読み込んで要約を作成します。

**構成の見本要約に含まれる内容:**
- 構成パターン（オープニング → 本編 → エンディング）
- 文体の特徴（語尾、リズム、句読点）
- 語り出しのパターン
- 転換表現

**ネタの台本要約に含まれる内容:**
- 情報の提示順序
- 情景描写のパターン
- 具体例の使い方

**例:**
```
3つのファイルを読み込んでいます...
- アカウント情報: sleep_travel
- 構成の見本: sleep_travel_greek_mythology
- ネタの台本: sleep_travel_universe_01

要約を作成しました。

【構成の見本要約の一部】
- 語尾: 「〜でございます」「〜でした」を基調
- 文の長さ: 中文中心（30-60文字）
- 語り出し: 「はるかなる昔〜」「かつて〜」など

【ネタの台本要約の一部】
- 情報の流れ: 起源 → 発展 → 現在
- 描写方法: スケール感を重視、数値で具体化
- 比喩表現: 身近なものとの比較を多用

この要約を元に台本を作成します。
```

### ステップ6: 要件確認

テーマや目標などを確認します。

**質問される内容:**
- 今回のテーマ（例: 北欧神話、宇宙の旅、エジプト文明）
- 台本の用途（YouTube動画、ポッドキャスト、etc）
- 目標（睡眠導入、教育、エンターテイメント）
- 制約（特定の禁止表現、必須キーワード、時間制限）

### ステップ7: 章構成の作成

スキルが全体の章構成を提案します。

**短尺の場合の例:**
```
以下の章構成でよろしいですか？

【章構成】
1. オープニング（1分）
2. 本編
   - 第1のポイント: ○○（3分）
   - 第2のポイント: △△（3分）
   - 第3のポイント: □□（3分）
3. エンディング（1分）

合計: 約11分
```

**長尺の場合の例:**
```
以下の章構成でよろしいですか？

【章構成】
1. オープニング（3分）
2. 本編
   - 第1話: ○○（12分）
   - 第2話: △△（12分）
   - 第3話: □□（12分）
   - 第4話: ◇◇（12分）
3. エンディング（BGMのみ）

合計: 約51分
```

→ 「はい」で承認、修正したい場合は具体的な要望を伝えてください

### ステップ8: 第1章の執筆

スキルが第1章を執筆します。

**執筆時に読み込むファイル:**
- ✅ アカウント情報
- ✅ 構成の見本要約
- ✅ ネタの台本要約
- ✅ 章構成

**例:**
```
第1章（オープニング）を執筆しています...

【台本】
ここは、寝ながらトラベルです。いつもご視聴いただきありがとうございます。このチャンネルは睡眠用の朗読チャンネルです。よろしければお休みになりながら、やお耳に時間がある時など、聞いてみてください。

すぐに本編をお聞きになりたい方、このオープニングを飛ばしたいという方は、概要欄にあるタイムスタンプをタップして、どうぞ飛ばしてお好きなところからお聞きください。

（中略）

それではゆっくりおやすみなさい。

【執筆情報】
- 文字数: 約900文字
- 推定時間: 約3分
- 狙い: 視聴者を落ち着かせ、今日のテーマを紹介

第1章が完成しました。次に進みます。
```

**重要**: 台本ファイルには見出しやメタ情報は含まれません。純粋に音声化されるテキストのみが保存されます。

### ステップ9: 自動進行

各章の完成後、確認待ちはしません。  
章を保存して提示したら「次に進みます」と通知し、次章を続けて執筆します。

### ステップ10: 第2章以降の執筆

同じ流れで、各章を一つずつ執筆していきます（自動進行）。

**毎回読み込むファイル（重要！）:**
- ✅ アカウント情報
- ✅ 構成の見本要約（構成パターンを忘れないため）
- ✅ ネタの台本要約（内容を忘れないため）
- ✅ 章構成（全体の流れを維持するため）
- ✅ 前章までの草稿（コンテキストとして）

### ステップ11: 最終統合と整理

全ての章が完成したら、スキルが自動的に統合・整理します。

#### 統合作業の流れ

1. **日本語タイトルの自動生成**
   - 台本の内容（オープニング部分など）を分析
   - メインテーマとサブタイトルを抽出
   - 日本語で簡潔なタイトルを作成
   - 形式: `[メインテーマ]_[サブタイトル]_[作成日YYYYMMDD].md`

2. **全章の統合**
   - セッションフォルダ内の全ての章ファイルを統合
   - 純粋な台本文のみを結合（見出し、メタ情報、タイムスタンプは追加しない）

3. **音声化向け分割整形（必須）**
   - 句読点（`。！？`）で1文ずつ分割
   - 長文は読点（`、`）や意味の切れ目で分割
   - 目安: 1文30-70文字、100文字超は必ず分割
   - 分割後の各行は単独で読んでも意味が通るようにする
   - `*_tts_chunks.md` を作成（1行1チャンク）

4. **最終台本の保存**
   - 日本語ファイル名で保存: `Remotion/scripts/voice_scripts/[日本語タイトル].md`
   - 最終成果物は `voice_scripts/` 側のみを保持

5. **作業ファイルの全削除（必須）**
   - `.sessions/[session_id]/` 配下のファイルは全削除
   - `【要約】` で始まる要約ファイルも全削除
   - `.sessions/[session_id]/` ディレクトリ自体も削除
   - 最終的に残すのは音声化に使う台本ファイルのみ

#### 例

**生成されるファイル名:**
```
# 台本
北欧神話_氷と炎の世界の始まり_20260211.md
宇宙の旅_星々の誕生と銀河の物語_20260212.md
エジプト神話_ファラオと神々の時代_20260213.md

```

**完了報告:**
```
全ての章が完成しました！
台本のタイトルを自動生成しています...

【タイトル】
エジプト神話_ファラオと神々の時代_20260211.md

最終台本を統合しています...
.sessions と要約ファイルを削除しています...

【完成】
台本: Remotion/scripts/voice_scripts/エジプト神話_ファラオと神々の時代_20260211.md
音声化用チャンク: Remotion/scripts/voice_scripts/エジプト神話_ファラオと神々の時代_20260211_tts_chunks.md

【統計情報】
- 総文字数: 27,500文字
- 推定時間: 約100分
- 構成: オープニング + 本編4話

【台本ファイルの内容】
- 純粋な台本テキストのみ（音声化される文章）
- 見出し、メタ情報、タイムスタンプは含まれていません
- そのまま音声化ソフトに渡せる形式です
- 長文分割済みの `*_tts_chunks.md` も同時に生成されます
- `.sessions` と `【要約】` ファイルは最終的に残りません

台本作成が完了しました！
```

#### タイトル命名のルール

- ✅ **日本語のみ**（英語は使わない）
- ✅ **メインテーマ**: 1-3語（例: 「北欧神話」「宇宙の旅」）
- ✅ **サブタイトル**: 4-8語（例: 「氷と炎の世界の始まり」）
- ✅ **作成日**: YYYYMMDD形式
- ❌ 避ける: 英語、長すぎるタイトル、記号の乱用

## 短尺と長尺の選び方

### 短尺（5-15分）を選ぶべき場合

✅ **SNSで拡散したい**（YouTube Shorts、Instagram、TikTok）
✅ **忙しい人向け**に情報を届けたい
✅ **テンポよく**複数のトピックを扱いたい
✅ **ハウツーやティップス**を紹介したい
✅ **視聴者の集中力**を最後まで保ちたい

**適したコンテンツ:**
- ハウツー（「○○の方法」）
- リスト形式（「5つの○○」）
- クイックティップス
- ニュース解説
- 商品レビュー

### 長尺（40-120分）を選ぶべき場合

✅ **睡眠導入**を目的とする
✅ **深く掘り下げたい**テーマがある
✅ **物語性**のあるコンテンツ
✅ **リラックス効果**を重視
✅ **落ち着いた長尺で届けたい**

**適したコンテンツ:**
- 神話（ギリシャ、北欧、日本、エジプトなど）
- 宇宙の旅
- 古代文明
- 美術史
- 自然科学（海洋、植物、鉱物など）

### 比較表

| 項目 | 短尺 | 長尺 |
|------|------|------|
| 時間 | 5-15分 | 40-120分 |
| テンポ | 速い | ゆったり |
| 文体 | 短文、簡潔 | 中文、丁寧 |
| 繰り返し | なし | なし |
| 用途 | SNS、教育 | 睡眠導入、朗読 |

詳細は [FORMAT_DEFINITIONS.md](FORMAT_DEFINITIONS.md) を参照

## セッション管理

### セッションとは？

台本作成の進行状況を保存する仕組みです。途中で中断しても、後から続きを再開できます。

### セッションの構造

```
.sessions/[session_id]/
├── session_info.json       # セッション情報（進行状況）
├── structure_summary.md    # 構成の見本要約
├── content_summary.md      # ネタの台本要約
├── chapter_plan.md         # 章構成
├── draft_chapter_01.md     # 第1章の草稿
├── draft_chapter_02.md     # 第2章の草稿
└── ...
```

### セッションの中断

自動進行中でも、ユーザーが「中断」と指示すればセッションが保存されます。

**例:**
```
第2章が完成しました。次に進みます。
（ここで「中断」と指示すると停止）

→ セッション情報を保存しました。
   次回、「前回の続きから」と言うと再開できます。
```

### セッションの再開

**コマンド:**
```
前回の続きから台本を作成してください。
```

または

```
セッションを再開してください。
```

**例:**
```
前回のセッションを見つけました。

【セッション情報】
- テーマ: エジプト神話
- 進行状況: 第2章まで完成
- 次の章: 第3話: オシリスとイシス

このセッションを再開しますか？

→ 「はい」で再開
```

### 複数のセッションがある場合

最新のセッションが自動的に選ばれますが、特定のセッションを指定することもできます。

**コマンド:**
```
セッション [session_id] を再開してください。
```

## よくある質問

### Q1: アカウント情報を追加するには？

**A:** `Remotion/script_resources/account_profiles/` に新しい `.md` ファイルを作成してください。

フォーマット例:
```markdown
account_name: my_account

[target_audience]
- 例: 20-35歳、在宅ワーカー

[brand_persona]
- 例: 誠実 / 実務的 / 親しみやすい

[tone_and_manner]
- 例: 断定しすぎない、短文中心

（以下、その他の項目...）
```

### Q2: 見本台本を追加するには？

**A:** `Remotion/script_resources/script_samples/` に新しい `.md` ファイルを作成してください。

既存の台本を分析した内容や、独自の構成パターンを記載します。

### Q3: 章の途中で構成を変更したい

**A:** 自動進行中に「中断」と指示し、具体的な変更内容を伝えてください。

**例:**
```
第2章の内容が長すぎるので、2つに分割してください。
```

スキルが章構成を更新し、以降はその構成に従います。

### Q4: 完成した台本を編集したい

**A:** 生成された台本ファイル（`Remotion/scripts/voice_scripts/` 内）を直接編集してください。

または、セッションの草稿ファイルを編集してから再統合することもできます。

### Q5: 台本の要約を見たい

**A:** このスキルでは要約ファイルは最終的に削除されます。  
必要な場合は、作業中に要約内容をその場で確認してください。

### Q6: 短尺と長尺を途中で変更できる？

**A:** 章構成の段階（ステップ6）までなら変更可能です。執筆が始まった後は難しいです。

新しくセッションを開始することをおすすめします。

### Q7: 同じテーマで複数の台本を作成したい

**A:** 可能です。毎回新しいセッションが作成され、日本語タイトルとサブタイトルが自動生成されるため、同じテーマでも別々の台本として管理されます。

**例:**
- `北欧神話_氷と炎の世界の始まり_20260211.md`（第1回）
- `北欧神話_オーディンとヴァルハラ_20260212.md`（第2回）
- `北欧神話_ラグナロクと世界の終焉_20260213.md`（第3回）

サブタイトルと作成日が異なるため、内容の違いが明確です。

### Q8: セッションを削除したい

**A:** 完了時に自動削除されるため、通常は手動削除不要です。中断セッションだけ必要に応じて削除してください。

**コマンド例:**
```bash
rm -rf .agent/skills/script-writing-accounts-aware/.sessions/20260211_143000_norse_mythology
```

### Q9: 台本の文字数を指定できる？

**A:** 要件確認の段階（ステップ5）で「約10,000文字で」と伝えてください。

スキルが章の長さを調整します。

### Q10: 複数人で同時に台本を作成できる？

**A:** セッションIDがタイムスタンプベースなので、同時に複数人が作業しても衝突しません。

ただし、同じファイルを同時に編集する場合は注意が必要です。

### Q11: タイトルが気に入らない。変更できる？

**A:** 可能です。

**方法1（生成前）**: 完了報告時に「タイトルを変更したい」と伝える。希望するタイトルを提案してください。

**方法2（生成後）**: 生成されたファイル名を手動でリネーム。

**例:**
```bash
# Before
エジプト神話_ファラオと神々の時代_20260211.md

# After
```

### Q12: 一章ごとのファイルを残したい

**A:** このスキルでは最終的に `.sessions` 配下を残しません。  
一章ごとのファイルを保持したい場合は、完了前に明示して別途保存してください。

### Q13: 日本語ファイル名が扱いにくい

**A:** 日本語ファイル名は人間にとって読みやすいですが、一部のシステムでは扱いにくい場合があります。

**対処法:**
- ファイル名を英語にリネーム（手動）
- または、要件確認時に「英語ファイル名で」と伝える（例外対応）

ただし、デフォルトは日本語です（内容が一目で分かるため）。

### Q14: 台本に見出しやメタ情報が必要な場合は？

**A:** 台本ファイルは純粋な音声化テキストのみです。見出しやメタ情報が必要な場合は：

**方法1**: セッションフォルダの `chapter_plan.md` を参照
- 章構成や見出しが記載されています

**方法2**: 手動で別ファイルを作成
- `[タイトル]_情報.md` などの名前で管理用ファイルを作成

**方法3**: 台本の冒頭にコメントとして記載
- ただし、音声化時には手動で削除が必要

### Q15: タイムスタンプはどうやって作る？

**A:** タイムスタンプは台本ファイルには含まれていません。

**作成方法:**
1. **手動計算**: 台本の文字数から推定時間を計算
2. **音声化後**: 実際の音声ファイルから正確なタイムスタンプを取得
3. **自動ツール**: 音声ファイル分析ツールを使用

推定時間は完了報告時に提示されますが、実際の音声化後に微調整が必要です。

### Q16: 見出しなしで章の区切りが分からない

**A:** 台本ファイル内では改行のみで区切られています。

**章の区切りを確認する方法:**
1. **セッションフォルダ**: `chapter_plan.md` で章構成を確認
2. **内容から判断**: オープニング終了後、本編が始まる箇所など
3. **改行パターン**: 章の区切りは通常2-3行の改行

**例**:
```
（オープニング終了）

それではゆっくりおやすみなさい。


（ここから本編第1話が始まる）

はるかなる昔、時の名前すら持たぬ遠き時代のこと。
```

## トラブルシューティング

### 問題: 見本台本が見つからない

**原因**: ファイル名が間違っているか、ファイルが存在しない

**解決策**:
1. `Remotion/script_resources/script_samples/` の内容を確認
2. ファイル名を正確に指定（拡張子 `.md` を含む）
3. 見本台本を追加する

### 問題: 章構成が承認されない

**原因**: 構成が期待と異なる

**解決策**:
具体的な修正要望を伝える

**例:**
```
第3話と第4話を入れ替えてください。
```

または

```
もっと短くして、3つの話にしてください。
```

### 問題: 文体がアカウントのトーンと合わない

**原因**: アカウント情報の記載が不十分、またはスキルの理解不足

**解決策**:
1. アカウント情報ファイルを詳細化
2. 修正要望を伝える（「もっと丁寧な言い回しで」など）
3. 見本台本を変更する

### 問題: セッションが再開できない

**原因**: 既に完了しており、`.sessions` が削除済み

**解決策**:
1. 完了済みなら新しいセッションを開始
2. 中断済みなら `.sessions/` にセッションが残っているか確認

### 問題: 台本が完成したが、ファイルが見つからない

**原因**: 保存先のパスが間違っている

**解決策**:
1. `Remotion/scripts/voice_scripts/` を確認
2. ファイル名で検索する

**コマンド例:**
```bash
find . -name "*egypt_mythology*"
```

### 問題: 長尺が短く感じる

**原因**: このスキルでは繰り返し（2周目）を作成しないため

**解決策**:
本編の各章を拡張するように指示して再生成する
