# LINEセールスシナリオ設計スキル

## 目的

LPから流入した見込み客に対して、LINEを使って信頼を築き、個別説明会・セミナーへ誘導し、クロージングセッション（1回目・2回目）で成約に導くシナリオを設計する。

## 出力ファイル（最大6つ）

| ファイル名 | 内容 |
|---|---|
| `{prefix}_LINE_シナリオ.md` | LINEステップ配信の全メッセージ設計 |
| `{prefix}_LINE_画像プロンプト.md` | リッチメニュー・各ステップのバナー画像プロンプト |
| `{prefix}_セミナー誘導.md` | セミナー・説明会への誘導メッセージ設計（任意） |
| `{prefix}_1回目クロージング台本.md` | 個別セッション1回目の台本 |
| `{prefix}_2回目クロージング台本.md` | 個別セッション2回目のクロージング台本 |

出力先: ユーザーが指定したフォルダ、または `{プロジェクト名}/sales-scripts/` 以下

## 必須フロー

### フェーズ0：スキル起動時の確認

スキルが呼ばれたら、まず以下を確認してからフェーズ1に進む。

```
LINEセールスシナリオ設計スキルを開始します。

どこまで作りますか？
① LINEシナリオのみ
② LINEシナリオ + クロージング1回目台本
③ LINEシナリオ + クロージング1回目 + 2回目台本（フルセット）

数字で選んでください。
```

### フェーズ2：ヒアリング（設計に必要な情報）

以下を一度にまとめてヒアリングする（一問一答にしない）。

```
設計に必要な情報をいくつか確認させてください。

1. LINEの登録経路はどこですか？
   例：LP登録、広告、Instagram誘導、など

2. 見込み客が登録した時点での温度感は？
   - 冷たい（まだ検討していない）
   - ふつう（興味はある）
   - 温かい（今すぐ相談したい）

3. 個別説明会・セミナーへの誘導は何日以内に行いたいですか？
   例：3日以内、1週間以内、など

4. LINEのトーン（話し方）はどれに近いですか？
   - 友達っぽい、カジュアル
   - 丁寧・信頼感重視
   - 熱量・背中を押すタイプ

5. 発信者（キャラクター）はどんな人ですか？
   例：元サラリーマン・副業で月30万達成・〇〇歳、など
   ※ない場合は「なし」でOK
```

回答を受け取ったら「ありがとうございます。設計に入ります。」と宣言してフェーズ3へ。

### フェーズ3：LINEシナリオ設計・出力

#### 設計原則

- **最初の3通は信頼構築**（売らない・押さない）
- **中盤は痛みの言語化と共感**（見込み客が「自分のことだ」と思う）
- **後半は希望と行動喚起**（次のステップへの自然な誘導）
- **各メッセージは200〜400文字**（長すぎると読まれない）
- **スタンプ・画像の指示も入れる**（送るべき素材がある場合）

#### LINEシナリオのテンプレート構造

```
【Day0 登録直後】
目的：第一印象・信頼の種まき
内容：歓迎 + 発信者の自己紹介 + 「これから届けるもの」の予告

【Day1】
目的：共感・ターゲットの悩みの言語化
内容：「こんな悩みありませんか？」系のメッセージ

【Day2】
目的：発信者のストーリー（自己開示）
内容：過去の失敗・転機・今の状況を簡潔に

【Day3】
目的：ベネフィット提示 + 行動喚起（説明会・セミナーへの誘導）
内容：「解決策があります」→ 「詳しく話したい」→ セミナー・個別相談の案内

【Day5〜7（フォロー）】
目的：未返信・未申込み者への再誘導
内容：ソフトなリマインドと別角度からの価値提示

【セミナー前日リマインド】
目的：当日参加率を上げる
内容：明日の確認 + 参加することのメリット再提示

【セミナー後〜個別相談前】
目的：個別セッションへのブリッジ
内容：「次のステップを一緒に決めませんか」的なメッセージ
```

#### 出力形式

以下のフォーマットで各メッセージを書く。

```markdown
## Day0｜登録直後

**目的：** 第一印象・安心感の醸成

**メッセージ：**
---
{名前}さん、登録してくれてありがとうございます！

〇〇の□□です。

（本文）
---

**ポイント：** （このメッセージで狙う感情・反応）
**次のメッセージまでの間隔：** 翌日
```

書き出したら `{prefix}_LINE_シナリオ.md` に保存し、ユーザーに「LINEシナリオを書き出しました。確認してください。」と報告する。

### フェーズ4：1回目クロージングセッション台本の設計・出力

#### 設計原則

1回目は「成約させない」のが正解。ゴールは「この人から話を聞きたい」「次回もっと知りたい」という感情を引き出すこと。

| フェーズ | 時間目安 | 感情目標 |
|---|---|---|
| ラポール構築 | 0〜10分 | 「安心・この人は違う」 |
| 現状ヒアリング（SPIN） | 10〜30分 | 「自分の悩みが言語化された」 |
| 発信者の実績・ストーリー | 30〜40分 | 「この人は本物だ」 |
| 解決策の片鱗（全部言わない） | 40〜50分 | 「もっと知りたい」 |
| 実践ワーク（簡単なもの） | 50〜55分 | 「できた・手応えがある」 |
| 販売を止める演出 | 55〜58分 | 「売ろうとしていない」 |
| 次回予告・次のアクション | 58〜60分 | 「次の話を聞きたい」 |

#### SPINヒアリングの設計（現状・問題・影響・解決ニーズ）

商品情報とターゲット情報を元に、以下を設計する。

```
S（状況質問）：相手の現状を把握する質問
   例：「今どんな状況ですか？」「〇〇はやったことありますか？」

P（問題質問）：悩み・不満を引き出す質問
   例：「一番うまくいっていないのはどこですか？」

I（影響質問）：問題が続いた場合の影響を考えさせる質問
   例：「このまま1年経ったら、どうなっていそうですか？」

N（解決ニーズ質問）：理想の状態を言語化させる質問
   例：「もし解決できたら、どんな状態になりたいですか？」
```

#### 出力形式

```markdown
# 感情設計型 1回目セッション台本
**「この人の話を聞かないとやばい」と思わせる構造**

## 感情の旅 設計図
（時間・感情フェーズ・相手の頭の中をテーブルで）

## 01｜ラポール構築（0〜10分）
### 目的
### スクリプト
### 対話ポイント

## 02｜現状ヒアリング（10〜30分）
...
（以降フェーズごとに続ける）
```

書き出したら `{prefix}_1回目クロージング台本.md` に保存してユーザーに報告する。

### フェーズ5：2回目クロージングセッション台本の設計・出力

#### 設計原則

2回目は「自分で決めた」という感覚で参加費を払ってもらう設計。押し売りゼロ。

| フェーズ | 時間目安 | 感情目標 |
|---|---|---|
| リコネクト（前回の感情を再点火） | 0〜10分 | 「あの気持ち、まだある」 |
| 市場・問題の構造化 | 10〜35分 | 「一人では構造的に無理だ」 |
| 解決策の全体提示 | 35〜50分 | 「これが必要なものだ」 |
| 将来性・安心感・差別化 | 50〜60分 | 「ここしかない」 |
| コミット確認（合格演出） | 60〜65分 | 「自分で決めた」 |
| 価格提示・枠組み替え | 65〜80分 | 「やらない方が高くつく」 |
| ネクストアクション | 80〜85分 | 「今日中に動く」 |

#### 出力形式

1回目台本と同じ構造。PHASE番号、スライドマップ（スライドがある場合）、各フェーズのスクリプト、断り文句への返し、チェックリスト、設計思想を含める。

書き出したら `{prefix}_2回目クロージング台本.md` に保存してユーザーに報告する。

### フェーズ6：リッチメニュー・画像プロンプトの設計・出力

LINEシナリオの設計が完了したら、続けて以下を設計する。

#### リッチメニューとは

LINE公式アカウントのトーク画面下部に常時表示されるタップ可能なメニュー。見込み客がいつでも行動できる導線として機能する。

#### リッチメニューの設計ルール

- **レイアウト**：2列×2行（4分割）または 大1＋小2（3分割）が扱いやすい
- **メインCTA**：一番大きい枠に「セミナーを予約する」を配置
- **補助CTA**：特典受け取り・よくある質問・お問い合わせ など
- **デザイン**：LPのデザイントーン（グリーン×白×黒）に統一する

#### リッチメニューの出力形式

```markdown
## リッチメニュー設計

### レイアウト
（図解：どのエリアにどのボタンを置くかをテキストで説明）

### 各エリアの設定

| エリア | ラベル | アクション | 表示テキスト |
|---|---|---|---|
| 左上（大） | セミナー予約 | URLリンク | 今すぐ予約する |
| 右上 | 特典受け取り | URLリンク | 特典を受け取る |
| 左下 | よくある質問 | テキスト送信 or URL | よくある質問 |
| 右下 | お問い合わせ | URLリンク | お問い合わせ |

### 画像仕様
- サイズ：2500×1686px（推奨）または 2500×843px（ハーフ）
- 各エリアにアイコン＋テキストを配置
```

#### 各ステップのバナー画像プロンプト設計ルール

各ステップで使用するバナー・カード画像について、以下を記述する。

- **画像の用途**（どのステップで使うか）
- **サイズ・比率**（LINE推奨サイズを記載）
- **掲載テキスト**（画像上に載せるコピー）
- **デザイン仕様**（背景色・フォント・レイアウトの指示）
- **AI生成用プロンプト**（Midjourney / DALL-E / Canva AI 向けの英語プロンプト）

#### 画像一覧と出力形式

シナリオで使用する全画像を以下の形式でまとめて出力する。

```markdown
## 画像プロンプト一覧

---

### 画像①｜Step 1-B セミナー告知バナー

**用途：** Step 1-B ボタンテンプレートのメイン画像
**サイズ：** 1040×1040px（正方形）
**比率：** 1:1

**掲載テキスト（画像上のコピー）：**
- メイン：「ChatGPT活用実践個別セミナー」
- サブ：「無料開催中 ｜ 10大特典プレゼント」
- CTA：「10秒で簡単予約」

**デザイン仕様：**
- 背景：白（#FFFFFF）または薄グリーン（#f0fdf4）
- アクセントカラー：グリーン（#22C55E）
- テキスト：太字・黒（#1D2130）
- レイアウト：上部にロゴ、中央にキャッチ、下部にCTA

**AI生成用プロンプト（Midjourney / DALL-E）：**
（英語で記述）
```

書き出したら `{prefix}_LINE_画像プロンプト.md` に保存してユーザーに報告する。

## 注意事項

- **商品情報が曖昧な場合は、推測で進めずに確認する**
- **ターゲットの悩みは、受け取った情報から必ず言語化して提示する**
- **価格・成約率・実績などの数字は、ユーザーから提供されたもののみ使う（でっち上げない）**
- **メッセージは「送れる状態」で書く（変数は{名前}など分かりやすい形式で）**
- **1回目と2回目の台本は、前のフェーズで引き出した情報を引き継いで設計する**
