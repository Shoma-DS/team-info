# script

## 目的
- パターン分析をもとにテーマ案と `script.md` を作る

## 入力
- `viral_patterns.md`
- Any user-provided theme, angle, or platform constraint

## 出力
- Topic candidates when the theme is not fixed
- `script.md`

## 標準構成
- `hook`
- `opening`
- `section 1`
- `section 2`
- `section 3`
- `cta`

## 標準ルール
- 基本の想定尺は 40〜60 秒
- フックでは最初に主張、緊張感、問いのいずれかを明確に出す
- 各セクションは短く切り、字幕化しやすくする
- CTA は保存、コメント、次視聴のどれか 1 つに寄せる

## 品質チェック
- フックの言い回しを全セクションで反復しすぎない
- 音声化しても自然に読める話し言葉にする
- 曖昧な埋め草より、はっきりした構造を優先する

## テンプレ差分
- `profile.yaml` を読む
- `phase_refs.script` があれば追加で読む
- 差分ファイルは文体や構成の違いだけを書く
