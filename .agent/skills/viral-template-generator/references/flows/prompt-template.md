# prompt-template

## 目的
- `viral_patterns.md` の解析結果をもとに、台本を量産するためのプロンプトテンプレートを生成・保存する
- テーマ分類ごとのプロンプトテンプレートを Markdown で作成し、再利用できる状態にする

## 入力
- `inputs/viral-analysis/output/[batch]/viral_patterns.md`（必須）
- テンプレート対象のジャンル/チャンネル（例: 転職、副業、投資）

## 出力
- `inputs/viral-analysis/output/[batch]/prompt_templates/[テーマ名].md`（テーマ数分）

---

## テーマ分類のルール

`viral_patterns.md` のフック・タイトルキーワード・台本構成パターンから、以下の4分類を基準に仕分けする。
ジャンルによってキーワードは変わるが、4分類は共通で使う。

| 分類 | 視聴心理 | 代表キーワード（転職例） |
|---|---|---|
| 警告系 | 損失回避・不安喚起 | 注意・危険・バレる・辞められない |
| あるある系 | 自分ごと化・共感 | あるある・特徴・こんな人・要注意 |
| 理由系 | 安心感と焦りの両立 | できない理由・逃げられない・辞められない |
| 方法系 | 有益情報の獲得感 | 見抜く・やり方・方法・する方法 |

4分類に収まらないパターンは「その他」として都度追加してよい。

---

## プロンプトテンプレートの構造（各ファイル共通）

各テーマファイルは以下のセクションで構成する:

```
1. frontmatter（theme / pattern / source_analysis）
2. このテーマの特徴（解析より）
3. プロンプト本文（````で囲んだコードブロック）
   - テーマ変数（量産ポイント）
   - 構成ルール（変えない）
   - 出力フォーマット
   - 制約まとめ（尺・話速・字幕・BGM・SFX）
4. 出力例（参考）
5. 量産のやり方（コピー→変数変更→AIに貼り付け→保存）
```

---

## 実行ステップ

### Step 1: パターン分析の読み込み

`viral_patterns.md` から以下を抽出する:

- フックタイプ（statement / question / visual）
- タイトルに頻出するキーワード
- 本編の構成型（列挙 / ストーリー / 比較 など）
- 標準尺・字幕文字数・話速

### Step 2: テーマ分類の確定

抽出結果を4分類（警告系 / あるある系 / 理由系 / 方法系）に当てはめる。
ジャンルに固有の分類があれば追加してよい。

### Step 3: プロンプトテンプレートの作成

各分類について `[テーマ名].md` を作成する。
テーマ変数には `{{}}` 形式で差し込み箇所を明示する。
制約まとめには `viral_patterns.md` の数値をそのまま転記する。

### Step 4: 出力例の作成

各テンプレートに具体的な出力例を1つ含める。
例は短め（5選程度）でよい。

### Step 5: 保存

```
inputs/viral-analysis/output/[batch]/prompt_templates/
├── 警告系.md
├── あるある系.md
├── 理由系.md
└── 方法系.md
```

---

## 品質チェック

- テーマ変数が `{{}}` 形式で明示されているか
- 制約まとめに `viral_patterns.md` の実測値が入っているか
- 出力例がテンプレートどおりのフォーマットになっているか
- 量産のやり方（コピー→変えて貼る）が1ステップで実行できるか

---

## テンプレ差分

- `profile.yaml` に `phase_refs.prompt_template` があれば追加で読む
- ジャンルごとのキーワードはテンプレート内の「出力例」だけ変える
- 構成ルール・制約まとめ・フォーマットは共通ロジックを崩さない
