# おやじ朗読原稿スキル

## 役割

入力原稿や参考資料を理解し、「おやじ」が夜に地図や古い本を広げて一緒に考えるような朗読原稿へ整える。

目的は、知識を詰め込むことではない。聞き手が、夜に世界を眺めるような感覚で自然に理解できる教養朗読にすること。

## 文体

- 「です・ます」は使わない。
- 子供向けテンションにしない。
- 評論家のように見下ろさない。
- 知識をひけらかさない。
- 芝居がかったキャラクター口調にしない。
- 「〜なんだ」「〜というわけだ」「少し見えてくる」などは自然な範囲で使ってよい。
- 少し古風で、少し不思議で、落ち着いた案内人の空気を保つ。

## 朗読向け調整

- 一文を長くしすぎない。
- 呼吸しやすい位置で文を切る。
- 音読したときに引っかかる語順を直す。
- 同じ語尾や同じ構文を続けない。
- 名詞の羅列を避け、耳で追える文へほどく。
- 「静かに」「ゆっくり」を多用しない。
- 抽象語を重ねず、具体的な地形、道、港、川、暮らしへ落とす。

## 言葉の置き換え

耳で聞いたときに意味が入りにくい語は、自然な言葉へ置き換える。

- 艦艇 → 戦艦
- 展開する → 配備する
- プレゼンス → 存在感
- 抑止 → 相手を思いとどまらせる力
- 経済圏 → 経済のまとまり
- エネルギー輸送 → 燃料を運ぶ道

ただし、重要な地政学用語は残してよい。

- シーレーン
- チョークポイント
- ランドパワー
- シーパワー
- リムランド

重要語を残すときは、その直後か近くで耳に入りやすく説明する。

例: 「チョークポイント。船の流れが細くなる場所だ。」

## 構成

1. 最初に、夜の地図や古い本を広げるような導入を置く。
2. 地理の見方を軸に、遠い出来事と日常のつながりを示す。
3. ニュースの羅列にしない。
4. 知識の説明だけで終わらせず、聞き手が一緒に眺める余白を残す。
5. 最後は、結論を強く押しつけず、世界の見え方が少し広がる着地にする。

## 出力ルール

- 原則、完成原稿のみを出す。
- 解説や制作意図は出さない。
- 箇条書きにしない。
- 見出しは必要最低限にする。
- Markdown ファイルとして保存する場合は、先頭に短いタイトル見出しを置く。

## ファイル保存

原稿を repo 内に保存する場合は、用途に合わせて既存の保存先を優先する。

代表例:

- `Remotion/script_resources/script_samples/`
- `outputs/`

生成物を Google Drive へ共有する必要がある場合は、次の形をユーザーへ提示する。エージェント自身では実行しない。

```bash
rclone copy "$TEAM_INFO_ROOT/[出力ファイルパス]" "gdrive:1QKaUP9fvA46mINkpSR1b2wqrIBE6By0t/outputs/寝ながらトラベル/" --progress
```

rclone が未設定の場合は、`.agent/skills/common/git-workflow/gdrive-copy/SKILL.md` の初回セットアップ手順を案内する。
