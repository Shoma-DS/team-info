# アカウント連動 段階的台本作成スキル

## 目的
- 最終統合時に長文を句読点単位で適切に分割し、音声化しやすい台本にする

## 参照フォルダ
- アカウント情報: `Remotion/script_resources/account_profiles/`
- 見本台本: `Remotion/script_resources/script_samples/`
- 最終台本（音声化入力）: `Remotion/scripts/voice_scripts/`
- セッション管理: `.agent/skills/remotion/script-writing-accounts-aware/.sessions/`

## 必須フロー（改善版）

台本作成を開始したら、必ず次の順序で進める。

### フェーズ1: 初期設定

#### 1. アカウント選択
- `Remotion/script_resources/account_profiles/` 内のファイル一覧を提示する
- ユーザーに「どのアカウント情報を使うか」を1つ選んでもらう
- アカウント情報を読み込む

#### 2. 短尺/長尺選択
- ユーザーに「短尺」か「長尺」かを選んでもらう
- **短尺**: 5-15分程度、1-3つのセクション、繰り返しなし
- **長尺**: 40-120分程度、3-4つのセクション（繰り返しなし）

#### 3. 台本構成の見本選択
- `Remotion/script_resources/script_samples/` 内のファイル一覧を提示する
- ユーザーに「どの台本を構成パターンの見本として使うか」を1つ選んでもらう
- **目的**: 構成パターン、文体、リズムを参考にする

#### 4. 台本のネタ選択
- `Remotion/script_resources/script_samples/` 内のファイル一覧を再度提示する
- ユーザーに「どの台本を内容の参考（ネタ）として使うか」を1つ選んでもらう
- **目的**: 内容、情報の提示方法を参考にする
- **注意**: 構成の見本と同じファイルを選んでも良いし、別のファイルを選んでも良い

#### 5. 3つのファイルを読み込む（自動）

1. **アカウント情報を読み込む**
   - ブランドトーン、禁止表現、オープニングテンプレートなど
   - セッションフォルダに `account_info.md` としてコピー保存

2. **構成の見本を読み込んで要約作成**
   - 選択された構成見本を読み込む
   - 以下の要素を抽出して要約を作成:
     - 構成パターン（オープニング、本編の流れ、エンディング）
     - 文体の特徴（語尾、リズム、句読点の使い方）
     - 語り出しのパターン
     - 情景描写のパターン
     - 転換表現のパターン
   - 要約を `.sessions/[session_id]/structure_summary.md` に保存

3. **ネタの台本を読み込んで要約作成**
   - 選択されたネタ台本を読み込む
   - 以下の要素を抽出して要約を作成:
     - 内容の提示方法
     - 知識の織り込み方
     - エピソードの構成
     - 具体例の使い方
   - 要約を `.sessions/[session_id]/content_summary.md` に保存

- ユーザーに「3つのファイルを読み込み、要約を作成しました」と報告

#### 6. 要件確認
- ユーザーにヒアリング:
  - 今回のテーマ（例: 北欧神話、宇宙の旅、エジプト文明）
  - 台本の用途（YouTube動画、ポッドキャスト、etc）
  - 目標（睡眠導入、教育、エンターテイメント）
  - 制約（特定の禁止表現、必須キーワード、時間制限）

#### 6. 章構成の作成
- アカウント情報、短尺/長尺、テーマ、要件に基づいて全体の章構成を作成
- **短尺の場合**:
  - オープニング（30秒-1分）
  - 本編（1-3つのセクション）
  - エンディング（30秒-1分）
- **長尺の場合**:
  - オープニング（2-3分）
  - 本編（3-4つのセクション、各10-15分）
  - エンディング（BGMのみ）
- 章構成を `.agent/skills/remotion/script-writing-accounts-aware/.sessions/[session_id]/chapter_plan.md` に保存
- ユーザーに章構成を提示し、承認を得る

### フェーズ2: 段階的執筆

#### 7. 第1章の執筆
- **必ず実行**: 以下のファイルを読み込む
  - アカウント情報ファイル
  - 構成の見本要約 (`.sessions/[session_id]/structure_summary.md`)
  - ネタの台本要約 (`.sessions/[session_id]/content_summary.md`)
  - 章構成 (`.sessions/[session_id]/chapter_plan.md`)
- 第1章（オープニングまたは本編第1話）を執筆
- 執筆内容を `.agent/skills/remotion/script-writing-accounts-aware/.sessions/[session_id]/draft_chapter_01.md` に保存
- ユーザーに提示し、「第1章を出力しました。次に進みます。」と通知

#### 8. 自動進行
- 章ごとのユーザー確認は行わない
- 各章を保存・提示したら、即座に次章へ進む
- 各遷移時に「次に進みます」と短く通知する

#### 9. 第2章以降の執筆
- **必ず実行**: 以下のファイルを毎回読み込む（ネタを忘れないため）
  - アカウント情報ファイル
  - 構成の見本要約 (`.sessions/[session_id]/structure_summary.md`)
  - ネタの台本要約 (`.sessions/[session_id]/content_summary.md`)
  - 章構成 (`.sessions/[session_id]/chapter_plan.md`)
  - 前章までの草稿（コンテキストとして）
- 次の章を執筆
- 執筆内容を `.sessions/[session_id]/draft_chapter_XX.md` に保存
- ユーザーに提示し、「次に進みます」と通知して継続

#### 10. 長尺時の補足
- 長尺でも本編の繰り返し（2周目）は作成しない
- 見本台本に繰り返しがあっても、今回の出力では採用しない

#### 11. 最終統合と整理
1. **タイトルの自動生成**
   - 台本の内容（特にオープニング部分）を読み取る
   - メインテーマとサブタイトルを抽出
   - 日本語で簡潔なタイトルを作成
   - 形式: `[メインテーマ]_[サブタイトル]_[作成日YYYYMMDD].md`
   - 例: `北欧神話_氷と炎の世界の始まり_20260211.md`

2. **全章の統合**
   - セッションフォルダ内の全ての章ファイル（`draft_chapter_XX.md`）を読み込む
   - 1つのファイルに統合
   - **純粋な台本テキストのみ**を統合（見出し、メタ情報、タイムスタンプは追加しない）
   - 章と章の間は改行のみで区切る

3. **最終台本の保存**
   - 統合した台本を `Remotion/scripts/voice_scripts/[日本語タイトル].md` に保存
   - 最終成果物は `voice_scripts/` 側のみを保持する

4. **作業用ファイルの全削除（必須）**
   - 以下は最終的に不要なので必ず削除する:
     - `.sessions/[session_id]/` 配下の全ファイル
     - `【要約】` で始まる要約ファイル
   - 完了時は `.sessions/[session_id]/` 自体も削除する
   - 目的: 残骸ファイルを残さず、音声化に必要な最終台本のみを残す

#### 12. 共有ストレージへの自動コピー（必須）
- 最終台本を共有ストレージにコピーする。
- コピー先は `TEAM_INFO_SHARED_ROOT` を優先し、未指定なら一般的な Google Drive / OneDrive 配下の `team-info/` を自動検出する。
- コマンド例:
```bash
python .agent/skills/common/scripts/team_info_runtime.py copy-to-shared "Remotion/scripts/voice_scripts/<タイトル>.md"
```
- コピー成功後、「共有ストレージにもコピーしました」と報告する。
- コピー失敗時はエラーを報告し、手動コピーのパスを案内する。

#### 13. 完了報告
- 「全ての章が完成しました！」と報告
- 最終台本のパスを提示（日本語ファイル名）
- 推定時間、総文字数、構成を要約して提示
- 削除された `.sessions` 配下ファイル数と要約ファイル数を報告
- **注意**: タイムスタンプやメタ情報は口頭報告のみ。台本ファイルには書かない

### フェーズ3: セッション管理

#### セッションの保存
- 各セッションは `.agent/skills/remotion/script-writing-accounts-aware/.sessions/[session_id]/` に保存
- session_id は `YYYYMMDD_HHMMSS_theme` の形式

#### セッションに含まれるファイル

**作業中:**
```
.sessions/[session_id]/
├── session_info.json          # セッション情報（日時、アカウント、テーマ、進行状況）
├── structure_summary.md       # 構成の見本の要約（作業中のみ）
├── content_summary.md         # ネタの台本の要約（作業中のみ）
├── chapter_plan.md            # 章構成
├── draft_chapter_01.md        # 第1章の草稿（作業中のみ）
├── draft_chapter_02.md        # 第2章の草稿（作業中のみ）
└── ...
```

**完成後:**
```
.sessions/[session_id]/ は削除される（ファイルを残さない）
```

#### セッションの再開
- ユーザーが「前回の続きから」と言った場合（完了前の中断セッションのみ）:
  1. 最新のセッションフォルダを確認
  2. `session_info.json` から進行状況を読み込む
  3. 次の章から再開

## システムプロンプト相当の固定ルール

以下は台本作成時に常に適用する。

### 役割
あなたはSNS/YouTube向け動画台本の作成者。アカウントのブランドを理解し、段階的に台本を執筆する。

### 最優先事項
1. **選択アカウントのブランド人格・トーン・禁止事項を厳守**
2. **構成の見本要約を毎回読み込み、構成パターンと文体を参考にする**
3. **ネタの台本要約を毎回読み込み、内容と情報提示方法を参考にする**
4. **章構成を毎回読み込み、全体の流れを維持する**
5. **一章ごとに執筆し、確認待ちせず自動で次章へ進む**
6. **丸写しは絶対禁止、3語以上の連続一致を避ける**

### 文章方針
- 冒頭でフックを入れる（短尺の場合は特に重要）
- 1文を短くし、音読しやすいリズムにする（長尺は中文でも可）
- 抽象語より具体語を優先する
- 行動喚起（CTA）を適切な位置に置く
- 長すぎる一文を避ける（句読点で自然に分ける）

### 禁止事項
- 断定できない効果の誇張
- アカウント情報に反する語調・表現
- 見本台本のコピー
- 章構成を無視した執筆
- 一気に全章を書く（必ず一章ずつ）

### 出力形式

**重要**: 台本は音声化されるテキストのみを出力する。メタ情報、見出し、注釈は一切含めない。

#### 各章の執筆時

**❌ 禁止（音声化されてしまう）**:
```
## 第○章: [タイトル]
## オープニング
### 第1話: [タイトル]
**執筆メモ**: ...
```

**✅ 正しい形式（純粋な台本テキストのみ）**:
```
[台本本文をそのまま書く]
```

#### 例

**❌ 悪い例**:
```
## オープニング

ここは、寝ながらトラベルです。いつもご視聴いただきありがとうございます。

---

**執筆メモ**: 約900文字、3分
```

**✅ 良い例**:
```
ここは、寝ながらトラベルです。いつもご視聴いただきありがとうございます。このチャンネルは睡眠用の朗読チャンネルです。よろしければお休みになりながら、やお耳に時間がある時など、聞いてみてください。

すぐに本編をお聞きになりたい方、このオープニングを飛ばしたいという方は、概要欄にあるタイムスタンプをタップして、どうぞ飛ばしてお好きなところからお聞きください。

（以下、台本が続く...）
```

#### 章の区切り

章の区切りは**改行のみ**で表現する。見出しは使わない。

**例**:
```
（オープニングの台本）

それではゆっくりおやすみなさい。


（本編第1話の台本がそのまま始まる）

はるかなる昔、時の名前すら持たぬ遠き時代のこと。
世界には、まだ何もございませんでした。

（以下続く...）
```

#### 執筆メモの扱い

執筆メモ（文字数、推定時間、狙いなど）は、ユーザーに口頭で報告する。台本ファイルには書かない。

**報告例**:
```
第1章が完成しました。

【執筆情報】
- 文字数: 約900文字
- 推定時間: 約3分
- 狙い: 視聴者を落ち着かせ、今日のテーマを紹介

次に進みます。
```

### 台本ファイルの原則

1. **音声化されるテキストのみ**
   - 台本ファイルには、音声化される文章だけを書く
   - ナレーターが読み上げる内容そのまま

2. **禁止事項（音声化されてしまう）**
   - ❌ 見出し（`## オープニング`, `### 第1話`など）
   - ❌ メタ情報（タイトル、作成日、推定時間など）
   - ❌ タイムスタンプ（`00:00 オープニング`など）
   - ❌ 注釈（`**執筆メモ**:`など）
   - ❌ 区切り線（`---`）
   - ❌ マークダウン記法（太字、リストなど）

3. **許可事項**
   - ✅ 純粋な台本テキスト
   - ✅ 改行（章の区切り、段落の区切り）
   - ✅ 句読点（。、！？など）

4. **見本台本の扱い**
   - 見本台本に見出しやメタ情報が含まれていても、それらは無視する
   - 純粋な台本テキスト部分のみを参考にする

5. **長文分割の原則（音声化対応）**
   - 1行1文を基本にする
   - 1文が長い場合は、`、` や意味の切れ目で複数行に分割する
   - 分割後の各行は、そのまま単独で読まれても意味が通る形にする
   - 分割記号（`/` など）は入れない

## 短尺と長尺の定義

### 短尺（5-15分）
- **用途**: SNS、YouTube短編、教育コンテンツ
- **構成**:
  - オープニング: 30秒-1分
  - 本編: 1-3つのセクション（各3-5分）
  - エンディング: 30秒-1分
- **特徴**:
  - テンポが速い
  - 繰り返しなし
  - フックが強い
  - 情報を凝縮
- **文体**: 短文中心、簡潔な表現

### 長尺（40-120分）
- **用途**: 睡眠導入、長編朗読、深い教育コンテンツ
- **構成**:
  - オープニング: 2-3分
  - 本編: 3-4つのセクション（各10-15分）
  - エンディング: BGMのみ
- **特徴**:
  - ゆったりとしたリズム
  - 情景描写が豊か
  - 催眠効果を意識
- **文体**: 中文中心、丁寧な表現

## 見本台本の要約作成ガイドライン

見本台本を読み込んだ後、以下の項目を抽出して要約を作成する:

### 抽出項目
1. **構成パターン**
   - オープニングの流れ
   - 本編のセクション分け方法
   - エンディングの型

2. **文体の特徴**
   - 語尾のパターン（「〜です」「〜でございます」など）
   - 文の長さ（短文/中文/長文の比率）
   - 句読点の使い方

3. **語り出しのパターン**
   - 各章の冒頭表現
   - 時間表現（「かつて〜」「ある時〜」など）

4. **情景描写のパターン**
   - 視覚的描写の方法
   - 聴覚的描写の方法
   - 五感表現の使い方

5. **転換表現**
   - 話題転換の言い回し
   - 時間経過の表現

6. **リズムとテンポ**
   - 繰り返しのパターン
   - 対句表現の使い方

### 要約の形式
```markdown
# 見本台本の要約

## 基本情報
- 見本台本名: [ファイル名]
- 分類: 短尺/長尺
- 推定時間: 約○分

## 構成パターン
[オープニング→本編→エンディングの流れ]

## 文体の特徴
- 語尾: [パターン]
- 文の長さ: [短文/中文/長文]
- 句読点: [使い方の特徴]

## 語り出しパターン
[具体例]

## 情景描写パターン
[具体例]

## 転換表現
[具体例]

## リズムとテンポ
[特徴]

## 活用時の注意
[丸写し禁止、独自性を出す方法]
```

## 最終台本のタイトル命名規則

最終統合時に、台本の内容を読み取って自動的に日本語タイトルを生成する。

### 命名形式

```
[メインテーマ]_[サブタイトル]_[作成日YYYYMMDD].md
```

### タイトル生成の手順

1. **台本の内容を分析**
   - オープニング部分を重点的に読み取る
   - 「今日のお話は〜」「今回のテーマは〜」などの記述を探す
   - 章構成のタイトルを参照

2. **メインテーマの抽出**
   - 台本の主題を1-3語で表現（例: 「北欧神話」「宇宙の旅」「エジプト文明」）
   - カタカナ、漢字、ひらがなを適切に使用
   - 英語は使わない（日本語に翻訳）

3. **サブタイトルの抽出**
   - 今回の台本の具体的な内容を4-8語で表現
   - 例: 「氷と炎の世界の始まり」「星々の誕生と銀河の物語」
   - 詩的で魅力的な表現を心がける

4. **作成日の追加**
   - YYYYMMDD形式（例: 20260211）
   - タイムゾーンは日本時間

### 命名例

#### 短尺の場合
```
北欧神話_3柱の神々入門_20260211.md
宇宙の不思議_ビッグバンから銀河まで_20260211.md
エジプト文明_ファラオの黄金時代_20260211.md
```

#### 長尺の場合
```
北欧神話_氷と炎の世界の始まり_20260211.md
宇宙の旅_星々の誕生と銀河の物語_20260211.md
ギリシャ神話_天地の始まりと神々の誕生_20260211.md
日本神話_古事記に綴られた国生みの物語_20260211.md
```

### タイトル生成時の注意点

#### 避けるべき表現
- ❌ 英語（Norse Mythology → 北欧神話）
- ❌ 長すぎるタイトル（30文字以内が望ましい）
- ❌ 記号の乱用（!、?、...など）
- ❌ 「第1回」「Part1」などのシリーズ番号（作成日で区別）

#### 推奨する表現
- ✅ 日本語で簡潔に
- ✅ 内容が一目で分かる
- ✅ 詩的で魅力的
- ✅ ファイル名として使える文字のみ

### 同じテーマで複数回作成する場合

作成日が異なるため、自動的に区別される。

**例:**
```
北欧神話_氷と炎の世界の始まり_20260211.md  （第1回）
北欧神話_オーディンとヴァルハラ_20260212.md  （第2回）
```

## エラーハンドリング

### 見本台本が見つからない場合
- ユーザーに「見本台本が見つかりませんでした。テンプレートから作成しますか？」と確認
- Yes → 基本テンプレートを使用
- No → 見本台本を追加してもらう

### セッションが中断した場合
- セッション情報を保存
- 次回起動時に「前回の続きがあります。再開しますか？」と確認

### 章構成の変更が必要な場合
- ユーザーが途中で構成変更を希望した場合
- 章構成ファイルを更新
- 以降の章はその新しい構成に従う

## 使用例

### 開始コマンド（ユーザー側）
```
台本を作成してください。
アカウント: sleep_travel
テーマ: エジプト神話
```

### スキルの応答フロー
1. 「アカウント: sleep_travel を選択しました」
2. 「短尺（5-15分）と長尺（40-120分）、どちらにしますか？」
3. → ユーザー: 「長尺」
4. 「見本台本を選択してください: [リスト表示]」
5. → ユーザー: 「sleep_travel_greek_mythology」
6. 「見本台本を読み込んで要約を作成しています...」
7. 「要約が完成しました。続いて、章構成を作成します」
8. 「以下の章構成でよろしいですか？ [章構成表示]」
9. → ユーザー: 「はい」
10. 「第1章（オープニング）を執筆します...」
11. 「第1章が完成しました。[内容表示] 次に進みます。」
12. （第2章以降も同様に自動進行）
13. 必要ならユーザーが「中断」と指示して停止
14. 「全ての章が完成しました！最終台本を統合します...」
15. 「台本が完成しました！ファイルパス: [パス表示]」
