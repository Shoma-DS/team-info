# 台本作成のテンプレートとフォーマット

このファイルは `script-writing-accounts-aware` スキルから参照されるテンプレート集です。

## 1. 見本台本の要約フォーマット

見本台本を分析する際は、以下のMarkdown形式で要約を作成してください。

```markdown
# 見本台本の要約

## 基本情報
- 見本台本名: [ファイル名]
- 分類: [短尺/長尺]
- 推定時間: 約[○]分

## 構成パターン
[オープニング→本編→エンディングの流れを記述]

## 文体の特徴
- 語尾: [例: 「〜です」「〜だ」]
- 文の長さ: [例: 短文多め、リズム感重視]
- 句読点: [特徴]

## 語り出しパターン
[各章の冒頭表現の例]

## 情景描写パターン
[視覚・聴覚・五感表現の例]

## 転換表現
[話題転換や時間経過の言い回し]

## リズムとテンポ
[繰り返しの有無、対句表現など]

## 活用時の注意
[丸写しにならないための工夫点など]
```

## 3. 最終台本のタイトル命名規則

全章統合時に自動生成するファイル名のルールです。

**形式**: `[メインテーマ]_[サブタイトル]_[作成日YYYYMMDD].md`

### 命名の手順
1. **メインテーマ**: 台本の主題（例: 北欧神話、宇宙の旅）。日本語で1-3語。
2. **サブタイトル**: 具体的な内容（例: 氷と炎の世界の始まり）。詩的で魅力的に、4-8語。
3. **作成日**: YYYYMMDD形式。

### 良い例
- `北欧神話_3柱の神々入門_20260211.md`
- `宇宙の旅_星々の誕生と銀河の物語_20260211.md`
- `日本神話_古事記に綴られた国生みの物語_20260211.md`

### 悪い例
- `Norse_Mythology_Part1.md` (英語、シリーズ番号は避ける)
- `神話について_20260211.md` (抽象的すぎる)
- `今日の台本.md` (内容が不明)

## 4. 執筆報告テンプレート（ユーザー確認用）

台本ファイル自体にはメタ情報を書きませんが、ユーザーへの報告時には以下の情報を伝えてください。

```text
第○章が完成しました。

【執筆情報】
- 文字数: 約[○]文字
- 推定時間: 約[○]分
- 狙い: [この章の意図]

（次に進みます / 完了しました）
```
