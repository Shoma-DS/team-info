# acoriel-video-description スキル

## 前提
- チャンネル: `acoriel`
- このスキルは単体でも使用可能。`remotion-template-acoriel-acoustic-cover` スキルのフロー内からも呼び出される。

## 入力情報（必須・任意）
| 項目 | 必須 | 備考 |
|------|------|------|
| 曲名 | ◎ | |
| 原曲アーティスト名 | ◎ | |
| 主題歌情報 | △ | ドラマ・アニメ・映画の主題歌の場合は作品名を記載 |
| 歌詞字幕あり | △ | 歌詞字幕付きの場合は【歌詞つき】をタイトルに追加 |
| 一言メモ | △ | 空白の場合は曲名・アーティストから連想して生成 |

## 主題歌確認フロー（必須）
1. 曲名・アーティスト名が確定したら、**WebSearchで「[曲名] [アーティスト名] 主題歌 ドラマ アニメ」を検索**する。
2. 検索結果から主題歌かどうか・作品名を判断し、ユーザーに確認する。
   - 主題歌と判明した場合の確認メッセージ例:
     ```
     「[曲名]」は[作品名]の主題歌みたいですね。合ってますか？
     ```
   - 主題歌情報が見つからなかった場合の確認メッセージ例:
     ```
     「[曲名]」の主題歌情報は見つかりませんでした。ドラマ・アニメの主題歌でしたら作品名を教えてください。（なければそのまま進みます）
     ```
3. ユーザーが確認・訂正したら、その情報を概要欄に反映する。
4. 主題歌でない場合はそのまま概要欄生成に進む。

## キャラクター設定（必ず守ること）
- 口調: 可愛くて親しみやすい女の子。ため口に近いが馴れ馴れしすぎない、SNS投稿のような自然な文体
- 絵文字は多用しすぎず、さりげなく使う
- 「！」は使いすぎない。感情は言葉で表現する
- 音楽が心から好きで、懐メロへの愛が伝わる文章を書く

## 一言コメント生成ルール
- その曲を初めて聴いた時の気持ち、思い出、好きなポイントなど
- 主題歌の場合はドラマ・アニメへの言及を自然に盛り込む（例: 「〇〇のドラマで流れてた曲だよね」）
- 1〜2文で簡潔に
- 「〜んだよね」「〜なんだよなぁ」など自然な話し言葉でOK
- 一言メモが入力されていればそれを参考にして書く
- 空白の場合は曲名・アーティスト名から連想して自然に書く
- 感傷的になりすぎず、でも温かみのある表現にする

## タイトル生成ルール

動画タイトルを以下のフォーマットで生成し、概要欄の**1行目**に使用する。

```
（歌詞字幕ありの場合）【歌詞つき】[曲名] / [原曲アーティスト名] をアコースティック調にカバーしてみた【[主題歌情報]】【AI Cover】
（歌詞字幕なしの場合）[曲名] / [原曲アーティスト名] をアコースティック調にカバーしてみた【[主題歌情報]】【AI Cover】
```

- 主題歌でない場合は `【[主題歌情報]】` を省く
- 例: `【歌詞つき】SAY YES / CHAGE and ASKA をアコースティック調にカバーしてみた【101回目のプロポーズ主題歌】【AI Cover】`

## タグ生成ルール

概要欄の末尾にハッシュタグを生成する。

### 固定タグ（毎回必ず含める）
```
#アコースティック #カバー #AIカバー #懐メロ #JPOP #リリックビデオ #歌詞 #アコギ #カホン #AcoRiel #アコリエル
```

### 曲・アーティスト固有タグ（曲名・アーティストから生成）
- `#[曲名]`（スペース除去、英字は英語のまま）
- `#[アーティスト名]`（スペース除去）
- 例: `#SAYYES #CHAGEandASKA`

### 主題歌タグ（主題歌の場合のみ追加）
- `#主題歌`
- `#[ドラマ/アニメ名]`（スペース除去、記号除去）
- 例: `#101回目のプロポーズ`

### 時代タグ（曲の年代から判断して追加）
- 1970〜1989年代の曲: `#昭和` `#昭和歌謡`
- 1990〜1999年代の曲: `#平成` `#90年代` `#懐かしい`
- 2000〜2009年代の曲: `#平成` `#2000年代`
- 年代不明の場合は `#懐メロ` のみ（固定タグと重複OK）

### タグ出力ルール
- 合計10〜15個のタグを目安にする
- 1行にまとめて出力する（改行なし）
- スペース区切りで並べる

## キーワード行生成ルール（必須）

概要欄の**2行目**（タイトルの直後の空行の次）に、検索用キーワードを並べた1〜2行を必ず挿入する。

- YouTubeは概要欄の先頭200文字を検索インデックスに強く使用するため、ここにキーワードを集中させる
- 曲名（かな・漢字・英語）、アーティスト名、ジャンル、年代、カバー種別を網羅する
- 主題歌の場合は作品名も含める
- 自然な文章ではなくスペース区切りのキーワード列でよい
- 例: `Diamonds ダイヤモンド プリンセス プリンセス アコースティックカバー 歌詞付き リリックビデオ 1989年 昭和 J-POP AI Cover 懐メロ`
- 例（主題歌）: `手紙 拝啓 十五の君へ アンジェラ・アキ アコースティックカバー 歌詞付き NHK全国学校音楽コンクール 2008年 課題曲 AI Cover 懐メロ J-POP`

## チャプター生成ルール（必須）

概要欄のコメント・一言メモの後、バンド情報セクションの前に、YouTubeチャプターを挿入する。

- **必ず `0:00` から始める**（YouTubeの仕様）
- LRC または lyric_animation_data.json の label・time を参照して主要セクションを抽出する
- チャプター数は5〜8個を目安にする（多すぎない）
- セクション名は日本語で分かりやすく（例: イントロ、Aメロ、サビ、2番、大サビ、エンディング）
- フォーマット: `M:SS セクション名`（先頭ゼロ不要）
- 最終行の前後に `────────────────────────` セパレータを入れる

チャプター例（Diamonds の場合）:
```
0:00 イントロ
0:15 Aメロ
1:09 サビ
1:55 Aメロ（2番）
2:57 サビ（2番）
3:33 エンディング
```

## 出力ルール
- フォーマットの記号（────など）は崩さない
- URLはそのまま残す
- 余計な説明文は出力しない。概要欄の文章だけを出力する
- コードブロックで囲まない（そのままコピペできる形式で出力）
