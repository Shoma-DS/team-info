# team-info 概要説明マニュアル（まずはこちらをお読みください）

このファイルは、`team-info` 全体の入口マニュアルです。  
今のフォルダ構成、運用ルール、よく使うスキルの役割をまとめています。

## 1. まず覚えること

- AI スキルの正本は `.agent/skills/` にあります。
- エージェント向けの全体ルールは `AGENTS.md`、命名規則と禁止事項は `RULES.md` が正本です。
- コマンドは固定の `/Users/...` ではなく、`TEAM_INFO_ROOT` から絶対パスを作って使います。
- 新しいパソコンでは、リポジトリルートで次を 1 回実行して作業場所を登録します。

```bash
python .agent/skills/common/scripts/team_info_runtime.py setup-local-machine --repo-root .
```

- このパソコンをオーナー機として使うときだけ、次のように `--owner` を付けます。

```bash
python .agent/skills/common/scripts/team_info_runtime.py setup-local-machine --repo-root . --owner
```

- Git の push 前のオーナー確認は、人に聞かず、`team_info_runtime.py owner-status` でこのパソコンを見て判断します。

## 3. 主要フォルダの役割

- `.agent/skills/`
  - AI スキル本体。`SKILL.md` と必要な `scripts/` や `assets/` が入ります。
- `inputs/`
  - 入力素材の置き場です。生成物は置きません。
- `outputs/`
  - 生成物の正本の出力先です。今後も出力先はここに固定します。
  - AcoRiel 動画: `outputs/acoriel/renders/`
  - AcoRiel 概要欄: `outputs/acoriel/descriptions/`
  - Sleep Travel 音声: `outputs/sleep_travel/audio/`
  - Sleep Travel 動画: `outputs/sleep_travel/renders/`
  - ジモティー: `outputs/jmty/factory/` と `outputs/jmty/remote/`
  - バズ動画解析: `outputs/viral-analysis/`
- `Remotion/`
  - 動画制作と音声生成の作業場所です。
  - 実際のレンダリング実装は `Remotion/my-video/` にあります。
- `マニュアル/`
  - 人が読むための入口です。正本のルールは `AGENTS.md` と `RULES.md` を参照します。

## 4. 出力先ルール

- 出力先は常にルートの `outputs/` を使います。
- `Remotion/output/` `Remotion/renders/` `Remotion/my-video/out/` `Remotion/my-video/outputs/` は使いません。
- AcoRiel の概要欄は `outputs/acoriel/descriptions/` に保存します。
- レンダリングコマンドは、できるだけ次を使います。

```bash
bash "$TEAM_INFO_ROOT/Remotion/scripts/render_to_outputs.sh" <CompositionId> <output-file>.mp4
```

- `npx remotion render` を直接使うときも、出力先は `"$TEAM_INFO_ROOT/outputs/<channel>/renders/"` に固定します。

## 6. 運用ルール

- 新規スキル追加時は、`AGENTS.md` のスキル一覧も更新します。
- 人向けマニュアルを直すときは、先に `AGENTS.md`、`RULES.md`、各 `SKILL.md` の内容を確認して合わせます。
- 入力は `inputs/`、生成結果は `outputs/` に分けて管理します。
- 大きい変更や削除、push の前は、必ずユーザー確認を取ります。
- ユーザーに選択や確認を求める前は、通知を鳴らします。
