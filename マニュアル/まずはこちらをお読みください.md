## 2. team-info の主要フォルダ
- `.agent/`
: エージェント設定とスキル本体（`SKILL.md`、`scripts/`、`assets/`）
- `inputs/`
: 投稿作成の入力データ
: `jmty_factory_cases/`（工場案件）
: `jmty_remote_samples/`（在宅見本・職種リスト）
- `outputs/`
: 生成物の統一出力先（カテゴリ別管理）
: `acoriel/renders/`・`acoriel/descriptions/`
: `sleep_travel/audio/`・`sleep_travel/renders/`
: `jmty/factory/<timestamp>/post01.md ... post12.md`
: `jmty/remote/<timestamp>/post01.md ... post12.md`
- `Remotion/`
: 台本・音声・動画制作の作業領域
- `マニュアル/`
: 参照用マニュアル集約フォルダ

## 2.2 Remotion 直下のフォルダ概要
- `configs/`
: 音声生成などの共通設定ファイル置き場。例: `voice_config.json`（話者・話速・音量など）。
- `scripts/`
: 実行用入力データの置き場。現在は `voice_scripts/` に音声化対象の台本を配置。
- `script_resources/`
: 台本作成時の参照資料。アカウント設定、見本台本、生成済み台本の管理に使用。
- `my-video/`
: Remotionプロジェクト本体。`src/` で実装、`public/assets/channels/` でチャンネル別素材を参照し、レンダリング対象のCompositionを管理。

## 2.3 Remotion 直下の主要ファイル概要
- `generate_voice.py`
: VOICEVOX連携で台本から音声を生成するメインスクリプト。
- `run.sh`
: 音声化フローの実行ラッパー。通常運用での起動入口。
- `setup.sh`
: 初回セットアップ用（仮想環境・依存関係などの準備）。

## 3. 現在のスキル一覧と機能概要
- `git-workflow`
: Gitの基本運用を安全に進める補助
- `macos-intel-compatibility`
: macOS Intel環境向けの互換対応手順
- `remotion-video-production`
: Remotion動画制作の親スキル（チャンネル/テンプレ選択）
- `remotion-unified-output-routing`
: 出力先を `outputs/` 配下に統一し、`sleep_travel` / `acoriel` / `jmty` で分類する運用スキル
- `remotion-template-sleep-travel-long-knowledge-relax`
: sleep_travel長尺テンプレ編集
- `remotion-template-sleep-travel-short-digest`
: sleep_travel短尺テンプレ編集
- `script-writing-accounts-aware`
: アカウント方針に沿った台本作成
- `voice-script-launcher`
: VOICEVOX音声化フロー実行
- `jmty-posts`
: ジモティー投稿作成の親スキル（工場/在宅の振り分け）
- `jmty-posts-factory-12`
: 工場求人投稿を12本作成（案件ファイル基準）
- `jmty-posts-remote-12`
: 在宅求人投稿を12本作成（見本ファイル基準）
- `jmty-posts-gdrive-sync`
: `outputs/jmty/factory` と `outputs/jmty/remote` をGoogle Driveへ上書き同期
- `jmty-posts-12-variants`（旧互換）
: 旧方式の12本作成スキル（親子スキル移行前）

## 4. 運用ルール
- 新規スキル追加時は、`マニュアル/skills/` にショートカットを追加する。
- 投稿データは `inputs/`（入力）と `outputs/`（生成結果）を分けて運用する。
- 工場/在宅の投稿はタイプ別フォルダで管理し、共有時は `jmty-posts-gdrive-sync` を利用する。
