# 開発の進め方

AI エージェント（Claude・Codex・Gemini など）を使った基本的な開発ステップです。

---

## ステップ 1 — 最新状態に更新する

```
/pull
```

作業を始める前に、チームの最新変更を手元に取り込みます。
これをやらないと、他の人の変更と衝突しやすくなります。

---

## ステップ 2 — 開発モードを設定する

チームと共有する作業か、自分だけのメモや試作かで切り替えます。

| コマンド | 使うとき |
|---------|---------|
| `/team` | チームに共有したい変更を作るとき |
| `/personal` | 自分だけのメモ・試作・設定など |

> **個人モードの場合:** 新規作成したファイル・フォルダは自動で `.gitignore` に追加され、git には残りません。

---

## ステップ 3 — 開発する

自由に作業してください。
エージェントに頼んでファイルを作ったり、コードを書いたりできます。

---

## ステップ 4 — 定期的にコミットする（こまめに保存）

```
/c
```

変更が一区切りついたらコミットして保存します。
プッシュはしないので、気軽にこまめに使えます。

| 機種 | 動作 |
|------|------|
| オーナー機 | main ブランチにコミット |
| それ以外 | アカウント名のブランチにコミット |

---

## ステップ 5 — 開発終了・チームに共有する

```
/git
```

作業が終わったら `/git` でプッシュ・プルリクエストまで一気に行います。

| 機種 | 動作 |
|------|------|
| オーナー機 | main に直接プッシュ |
| それ以外 | アカウント名ブランチにプッシュ → プルリクエスト作成 |

> 未コミットの変更が残っている場合はコミットから自動で行います。
> すでにコミット済みの場合はプッシュのみ行います。

---

## ステップ 6 — Discord で報告する

プルリクエストを作ったら、Discord の以下の場所で報告してください。

- **サーバー:** team-info
- **カテゴリ:** バイブコーディング
- **チャンネル:** `#team-info開発メモ`
- **メンション:** `@SHO`

**報告メッセージの例:**
```
@SHO プルリクエスト出しました！
[変更の内容を一言で]
https://github.com/... ← PR のリンク
```

---

## まとめ（コマンド一覧）

```
/pull     → 最新を取り込む
/team     → チーム開発モードに切り替え
/personal → 個人開発モードに切り替え
/c        → コミット（こまめに保存）
/git      → プッシュ・PR 作成（開発終了時）
```
