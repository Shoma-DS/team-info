# 開発の進め方

AI に頼んで作業するときの、基本的な流れです。
コマンドを入力するだけで AI が動いてくれます。

---

## 全体の流れ

```
最新を取り込む → モードを決める → 作業する → こまめに保存 → チームに送る → 報告する
```

---

## ① 最新の状態にする

```
/pull
```

チームが作った最新の変更を自分のパソコンに取り込みます。
**作業を始める前に必ずやること。** やらないと他の人の変更と衝突することがあります。

---

## ② モードを決める

チームに共有するか、自分だけにするかを決めます。

```
/team      ← チームに共有したいとき
/personal  ← 自分だけのメモや試作のとき
```

> 個人モード（`/personal`）にすると、新しく作ったファイルは自動でチームから隠されます。

---

## ③ 作業する

AI に話しかけて、ファイルを作ったりコードを書いてもらいます。

---

## ④ こまめに保存する

```
/c
```

区切りがついたら保存します。チームには送りません。
失敗しても戻せるように、**こまめに使いましょう。**

| パソコンの種類 | どこに保存されるか |
|-------------|----------------|
| オーナー機 | main（メインの場所） |
| それ以外 | 自分の名前のブランチ（保存場所） |

---

## ⑤ チームに送る（作業が終わったとき）

```
/git
```

作業が終わったらチームに変更を送ります。

| パソコンの種類 | 何が起きるか |
|-------------|------------|
| オーナー機 | そのままチームの main に送られる |
| それ以外 | 自分の名前のブランチに送られて、プルリクエスト（変更のお知らせ）が作られる |

> まだ保存していない変更がある場合は、自動でコミット（保存）してから送ります。

---

## ⑥ Discord で報告する

プルリクエストを作ったら、Discord で報告します。

- **場所:** バイブコーディング → `#team-info開発メモ`
- **@SHO にメンション**してプルリクエストのリンクを貼る

**メッセージの例:**
```
@SHO プルリクエスト出しました！
〇〇の機能を追加しました
https://github.com/... ← リンク
```

---

## まとめ

| コマンド | タイミング |
|---------|----------|
| `/pull` | 作業を始める前 |
| `/team` か `/personal` | モードを決めるとき |
| `/c` | 区切りがついたとき（こまめに） |
| `/git` | 作業が終わったとき |
