# 生成された台本フォルダ

このフォルダには、`script-writing-accounts-aware` スキルを使用して生成された台本が保存されます。

## フォルダ構造

```
generated_scripts/
├── README.md （このファイル）
├── sleep_travel_norse_mythology_01.md （北欧神話編 第1回 - v1.0形式）
├── sleep_travel_universe_01.md （宇宙の旅編 第1回 - v1.0形式）
│
├── 北欧神話_氷と炎の世界の始まり_20260211.md （台本 - v2.0形式）
├── 【要約】北欧神話_氷と炎の世界の始まり_20260211.md （統合要約 - v2.0形式）
│
└── （今後生成される台本ファイルと統合要約ファイル）
```

## ファイル命名規則（v2.0）

生成される台本ファイルと統合要約ファイルは、**日本語タイトルが自動生成**されます。

### 命名形式

**台本:**
```
[メインテーマ]_[サブタイトル]_[作成日YYYYMMDD].md
```

**統合要約:**
```
【要約】[メインテーマ]_[サブタイトル]_[作成日YYYYMMDD].md
```

**台本:**

  - サブタイトル: 氷と炎の世界の始まり
  - 作成日: 2026年2月11日

  - メインテーマ: 宇宙の旅
  - サブタイトル: 星々の誕生と銀河の物語
  - 作成日: 2026年2月11日

**統合要約:**
- `【要約】北欧神話_氷と炎の世界の始まり_20260211.md`
- `【要約】宇宙の旅_星々の誕生と銀河の物語_20260211.md`

#### 短尺

**台本:**
- `北欧神話_3柱の神々入門_20260212.md`
- `エジプト文明_ファラオの黄金時代_20260212.md`

### タイトル生成のルール

- ✅ **日本語のみ**（英語は日本語に翻訳）
- ✅ **簡潔**: メインテーマは1-3語、サブタイトルは4-8語
- ✅ **内容が分かる**: 一目で何の台本か分かる
- ✅ **詩的**: 魅力的な表現
- ❌ **避ける**: 英語、長すぎるタイトル、記号の乱用

これらは参考として保持していますが、新しく生成される台本は日本語タイトル形式になります。

## 台本ファイルの構造（v2.0）

### 重要な変更点

**v2.0以降、台本ファイルは純粋な音声化テキストのみを含みます。**

### 含まれるもの ✅
- 音声化される台本テキスト（そのまま）
- 改行（章の区切り、段落の区切り）
- 句読点

### 含まれないもの ❌
- 見出し（`## オープニング`, `### 第1話`など）
- メタ情報（タイトル、作成日、推定時間）
- タイムスタンプ（`00:00 オープニング`など）
- 制作ノート（`**執筆メモ**:`など）
- マークダウン記法（太字、リストなど）

### 理由

台本ファイルは**音声化ソフトに直接渡す**ことを想定しています。
見出しやメタ情報が含まれていると、それらも音声化されてしまうため、純粋な台本テキストのみを保存します。

### 例

#### ❌ 旧形式（v1.0）

```markdown
# 【睡眠導入・眠くなる朗読・途中広告なし】北欧神話① 〜氷と炎の世界の始まり〜

## メタ情報
- タイトル: ...
- 推定時間: ...

## タイムスタンプ
00:00 オープニング
03:30 第1章
...

## 台本

### オープニング

ここは、寝ながらトラベルです。...

---

**執筆メモ**: 約900文字、3分

### 本編

#### 第1話: ギンヌンガガップ

はるかなる昔、...
```

#### ✅ 新形式（v2.0）

```
ここは、寝ながらトラベルです。いつもご視聴いただきありがとうございます。このチャンネルは睡眠用の朗読チャンネルです。よろしければお休みになりながら、やお耳に時間がある時など、聞いてみてください。

すぐに本編をお聞きになりたい方、このオープニングを飛ばしたいという方は、概要欄にあるタイムスタンプをタップして、どうぞ飛ばしてお好きなところからお聞きください。

（中略）

それではゆっくりおやすみなさい。


はるかなる昔、時の名前すら持たぬ遠き時代のこと。世界には、まだ何もございませんでした。

（以下、台本が続く...）
```

### 章の区切り

章の区切りは**改行のみ**で表現されます。

**例**:
```
（オープニング終了）

それではゆっくりおやすみなさい。


（本編第1話が始まる）

はるかなる昔、時の名前すら持たぬ遠き時代のこと。
```

通常、章と章の間は2-3行の改行が入ります。

### 統合要約ファイルとは？

v2.0以降、台本作成時に使用した2つの要約（構成の見本要約とネタの台本要約）を1つのファイルに統合して保存します。

**ファイル名:**
```
【要約】[メインテーマ]_[サブタイトル]_[作成日YYYYMMDD].md
```

**内容:**
```markdown
# 【要約】北欧神話_氷と炎の世界の始まり

## 構成の見本要約
[構成パターン、文体、リズムの要約]

## ネタの台本要約
[内容、情報提示方法の要約]
```

**用途:**
- 台本作成時に参考にした構成パターンと内容を記録
- 後から「この台本はどの見本を参考にしたか」を確認できる
- シリーズ物を作成する際に、前回と同じスタイルを維持しやすい

**例:**
```
【要約】北欧神話_氷と炎の世界の始まり_20260211.md
【要約】宇宙の旅_星々の誕生と銀河の物語_20260212.md
```

台本ファイルと統合要約ファイルは常にペアで保存されます。

### メタ情報やタイムスタンプはどこに？

メタ情報やタイムスタンプは台本ファイルには含まれていませんが、以下の場所で確認できます：

1. **セッションフォルダ**: `.agent/skills/script-writing-accounts-aware/.sessions/[session_id]/`
   - `chapter_plan.md`: 章構成
   - `session_info.json`: セッション情報（総文字数、推定時間など）

2. **完了報告**: 台本作成完了時に口頭で報告されます

3. **手動作成**: 必要に応じて別ファイルで管理してください

## 使い方

### 1. 台本の生成

Claude Codeで以下のように依頼します：

```
台本を作成してください。
アカウント: sleep_travel
テーマ: [あなたのテーマ]
```

### 2. 生成された台本の確認

生成された台本ファイルを開いて、内容を確認します。

### 3. 必要に応じて編集

- 固有名詞の表記確認
- 情報の正確性チェック
- 朗読のリズム確認
- タイムスタンプの調整

### 4. 朗読・録音

- 台本を読み上げて録音
- ゆっくりとしたペース（通常の0.8-0.9倍速程度）
- 句読点でしっかり間を取る

### 5. 動画制作

- BGM追加
- 必要に応じて映像追加
- タイムスタンプの最終確認

## 注意事項

### 著作権
- 生成された台本は、参考文献の内容を丸写ししていません
- 知識は事実（著作権の対象外）として扱い、表現は独自のものです
- 神話や科学的事実は基本的にパブリックドメインです

### 品質管理
- 生成された台本は必ず内容を確認してください
- 特に固有名詞、数値、科学的事実の正確性をチェック
- 朗読してみて、リズムや流れを確認

### バックアップ
- 重要な台本は別途バックアップを取ることを推奨
- Gitで管理している場合は、適宜コミット
