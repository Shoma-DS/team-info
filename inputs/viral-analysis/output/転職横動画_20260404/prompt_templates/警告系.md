# プロンプトテンプレート：警告系（会社批判・構造問題・辞めたくなる環境）

## このテーマの特徴（解析より）

- タイトルキーワード: 状況断定ワード（`そりゃ辞めるわ` `仕組みが終わってる` `逆効果すぎる`）＋ 会社/環境の特徴9選
- フック型: statement（状況を断定して即番号コール）
- 構成: タイトルコール → 9選列挙（各60〜90秒）→ 「こういう会社は変わらない」締め（全体8〜14分）
- 視聴心理: 「自分の会社のことだ」という認識から「やっぱり辞めるべきかも」への流れ
- 文体: 会社・組織の構造を観察型で描写。怒りを煽らず、「こういう仕組みになっている」と解説する
- バッチ代表作: 【そりゃ辞めるわ】社員が黙って去る会社の特徴9選 / 【仕組みが終わってる】社員を消耗させ続ける会社の特徴9選

## 構成ルール（変えない）

### タイトルコール（0〜15秒 / 2〜3文）
- タイトルをそのまま読み上げる
- 「こういう会社の特徴を9つ話します」のように即本題に入る
- 「今日は〜について話します」などの前置きは不要

### 本編（15秒〜終盤 / {{リスト数}}個 / 1項目60〜90秒）
各項目は以下の4ステップで書く:
1. **見出し**（1文）: 「1 〇〇」のような短いタイトル（会社の行動・構造を主語に）
2. **状況説明**（2〜3文）: 「〜という会社では…」「本来であれば〜のはずが〜」の対比
3. **構造解釈**（2〜3文）: 「なぜそうなるかというと〜」「この仕組みの問題は〜」で会社側の構造を解説
4. **社員目線の補足**（1〜2文）: 「社員からすると〜なのです」「そういう環境では〜になっていきます」

- 感情的な批判ではなく、「こういう仕組みになっている」という観察口調にする
- 社員の被害感情を煽らず、構造的な問題として描写する
- 「本来であれば〜」「現場の実態は〜」の対比構造を意識する

### 締め（最後 20〜30秒 / 2〜3文）
- 「こういう会社は仕組みが変わらない限り、社員の状況は変わらないことが多いです」などで締める
- 「だから転職しろ」と直接言わず、「自分の状況を改めて見直してみてください」程度に留める
- CTAは「保存して見返して」または「コメントで教えて」の1つだけ

**台本（冒頭2項目のみ）:**

【タイトルコール】
社員が黙って去る会社の特徴、9選です。
では、1つ目。

【本編】

1 現場の意見が反映されない
こういう会社では、会議で毎回意見を求められます。
しっかり資料を作って持っていくと、話を聞いてもらえる時間は数分で、また検討するという言葉が返ってきます。
本来であれば、意見を求めるという行為にはそれをしっかり検討して返答するという責任が伴うはずです。
しかしそれが繰り返されていくと、社員は意見を言っても変わらないと学習していくのです。

2 頑張りが評価に結びつかない
繁忙期に今回は頑張ってくれと言われて無理をしても、評価には何も反映されなかった。
そういう経験が続くと、頑張る理由がなくなっていきます。
会社側は意図して評価しないわけではなく、評価の仕組みそのものが機能していないケースが多くあります。
社員からすると、どれだけ動いても同じだという感覚になっていくのです。

【締め】
こういった特徴が重なっている職場では、仕組みが変わらない限り、状況は変わらないことが多いです。
ぜひ保存して、自分の環境を改めて見直す機会にしてください。
